# Математическая часть

## Архитектура модели

Модель представляет собой классический однослойный перцептрон (линейный классификатор). Для каждого входного вектора признаков $X$ вычисляется линейная комбинация с весами $w$ и смещением $b$:

$$z = Xw + b$$

Разделяющая граница определяется уравнением:

$$w_1x_1 + w_2x_2 + \dots + w_nx_n + b = 0$$

---

## Математический базис и оптимизация

### 1. Бинарная кросс-энтропия (BCE)

При  `loss_type="bce"` выход модели преобразуется в вероятность принадлежности к положительному классу с помощью логистической функции (сигмоиды):

$$\hat{y} = \sigma(z) = \frac{1}{1 + e^{-z}}$$

Для предотвращения вычислительного переполнения при расчете экспоненты в коде применяется численно стабильная версия сигмоиды.

Целевая функция потерь на батче размером $m$ с учетом $L_2$-регуляризации имеет вид:

$$\mathcal{L}_{BCE} = -\frac{1}{m}\sum_{i=1}^{m}\left[y^{(i)}\log\hat{y}^{(i)} + (1-y^{(i)})\log(1-\hat{y}^{(i)})\right] + \frac{\lambda}{2} \|w\|_2^2$$

Градиенты функции потерь по параметрам модели, используемые для их обновления на каждом шаге:

$$\frac{\partial \mathcal{L}_{BCE}}{\partial w} = \frac{1}{m} X^T (\hat{y} - y) + \lambda w$$

$$\frac{\partial \mathcal{L}_{BCE}}{\partial b} = \frac{1}{m} \sum_{i=1}^{m} (\hat{y}^{(i)} - y^{(i)})$$

---

### 2. Hinge Loss

При `loss_type="hinge_loss"` модель оптимизируется в режиме максимизации зазора. Исходные метки классов $y \in \{0, 1\}$ преобразуются во внутреннее представление $\tilde{y} \in \{-1, 1\}$ по правилу:

$$\tilde{y} = 2y - 1$$

При вычислении истории потерь в методе `hinge_loss` для обеспечения численной стабильности значения линейного прогноза $z$ предварительно ограничиваются (клипаются) в диапазоне $[-1 + 10^{-15}, 1 - 10^{-15}]$:

$$\tilde{z} = \text{clip}(z, -1 + 10^{-15}, 1 - 10^{-15})$$

Полная функция потерь на батче размером $m$ с учетом $L_2$-регуляризации имеет вид:

$$\mathcal{L}_{Hinge} = \frac{1}{m}\sum_{i=1}^{m} \max(0, 1 - \tilde{y}^{(i)} \tilde{z}^{(i)}) + \frac{\lambda}{2} \|w\|_2^2$$

В процессе обучения (в цикле оптимизации метода `fit`) расчет градиентов происходит на основе несрезанного линейного выхода $z$. Сначала определяется вектор маски `mask` на основе индикаторной функции:

$$\text{mask}^{(i)} = 1, \quad \text{если } \tilde{y}^{(i)} z^{(i)} < 1$$

$$\text{mask}^{(i)} = 0, \quad \text{в остальных случаях}$$

Затем вычисляется вектор градиентных коэффициентов `grad coeff`:

$$\text{grad coeff}^{(i)} = - \tilde{y}^{(i)} \cdot \text{mask}^{(i)}$$

Итоговые градиенты для модификации весов и сдвига по мини-батчу размером $m_b$ формируются в матричном виде следующим образом:

$$\frac{\partial \mathcal{L}_{Hinge}}{\partial w} = \frac{1}{m_b} X^T \cdot \text{grad coeff} + \lambda w$$

$$\frac{\partial \mathcal{L}_{Hinge}}{\partial b} = \frac{1}{m_b} \sum_{i=1}^{m_b} \text{grad coeff}^{(i)}$$

---

### 3. $L_2$-регуляризация

Для предотвращения эффекта переобучения (overfitting) и подавления избыточного роста весов, в модель интегрирована $L_2$-регуляризация. 

В целевую функцию потерь добавляется штрафное слагаемое, пропорциональное квадрату евклидовой нормы вектора весов. При расчете истории лоссов в конце каждой эпохи полная величина регуляризационного штрафа вычисляется по формуле:

$$\Omega(w) = \frac{\lambda}{2} \|w\|_2^2 = \frac{\lambda}{2} \sum_{j=1}^{n} w_j^2$$

Где $\lambda$ (`lambda_reg`) — управляющий гиперпараметр регуляризации. Смещение ($b$) традиционно исключается из процесса регуляризации, так как оно не влияет на ориентацию и кривизну разделяющей гиперплоскости.

В процессе обучения (в методе `fit`) вычисление частной производной функционала ошибки по весам на каждом мини-батче дополняется аддитивной составляющей градиента штрафа:

$$\frac{\partial \mathcal{L}_{reg}}{\partial w} = \frac{\partial \mathcal{L}_{0}}{\partial w} + \lambda w$$

Где $\mathcal{L}_{0}$ — градиент базовой функции потерь (BCE или Hinge Loss), вычисленный по текущему мини-батчу. 

В терминах шага оптимизации это приводит к модификации весов по правилу (эффект затухания весов / Weight Decay):

$$w^{(new)} = w^{(old)} - \eta \left( \frac{\partial \mathcal{L}_{0}}{\partial w} + \lambda w^{(old)} \right) = (1 - \eta \lambda) w^{(old)} - \eta \frac{\partial \mathcal{L}_{0}}{\partial w}$$

Благодаря этому на каждой итерации градиентного шага веса неявно уменьшаются в направлении нуля пропорционально их текущему значению, что изолирует модель от появления аномально больших коэффициентов и стабилизирует разделяющий зазор.

---

### Инициализация параметров

В перцептроне реализовано три режима начального распределения весов (при $b = 0.0$):
1. `zeros`: Матрица весов заполняется нулями.
2. `random`: Веса инициализируются случайными значениями из стандартного нормального распределения с малым масштабом: $w \sim \mathcal{N}(0, 0.01^2)$.
3. `large`: Веса инициализируются из нормального распределения с большой дисперсией: $w \sim \mathcal{N}(0, 10^2)$.

## Структура репозитория

* `perceptron.py` — класс `Perceptron`, содержащий логику прямого прохода, вычисления стабильной сигмоиды, подсчета лоссов (BCE / Hinge) и обучения через градиентный спуск.
* `data_prepare.py` — модули генерации синтетических данных (`generate_linear_gauss`, `generate_data`, `generate_circle`, `generate_xor`), разбиения выборки на обучающую/тестовую и стандартизации признаков.
* `utilities.py` — функции для расчета метрик эффективности (Accuracy, Precision, Recall, F1-Score, ROC AUC, Confusion Matrix), а также визуализации.
* `tests.py` — пайплайн `run_experiment` для изоляции конфигураций, замера времени вычислений и сохранения графических артефактов.
* `main.py` — центральный скрипт для последовательного запуска серии экспериментов.
* `graphics` - папка со всеми графиками
